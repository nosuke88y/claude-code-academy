## カリキュラム全体の構造的問題（最優先）

### [高] 1. Git/GitHubのセットアップ手順が全30レッスンに存在しない
- **指摘元**: A + B + C 全エージェント一致
- **問題**: Node.jsのインストールはCh.2で丁寧に扱うが、Gitのインストール手順とGitHubアカウント作成がどこにもない。Ch.4以降のハンズオン（コミット、プッシュ、PR作成、Vercelデプロイ）がすべて成立しない
- **修正案**: Ch.4-L1の冒頭に「Gitのインストール」セクションを追加（Windows: git-scm.com / Mac: `xcode-select --install` / 確認: `git --version`）。加えて「GitHubアカウント作成」「`gh auth login`」の手順をCh.4-L1またはL4に追加

### [高] 2. Claude Codeの料金・プラン説明がゼロ
- **指摘元**: A + C
- **問題**: 初心者が最も気にする「いくらかかるの？」への回答がコース全体で欠落。認証後に課金されたら信頼を失う
- **修正案**: Ch.2-L2（インストールと認証）に「料金について」セクション追加。「Claude Codeを使うにはAnthropicの有料プラン（Max Plan）が必要です」等を明記

### [高] 3. `$` マークの意味が全レッスンで未説明
- **指摘元**: A
- **問題**: 全コマンド例で `$ node -v` のように `$` が付くが、初心者は `$` ごと入力してしまう
- **修正案**: Ch.2-L1（最初にターミナルコマンドが出るレッスン）の冒頭に「※ `$` はターミナルの入力を示す記号です。実際に入力するのは `$` の後ろの部分だけです」と注記

### [高] 4. Webhook URLのセキュリティ注意書きが不足
- **指摘元**: B
- **問題**: L28（Slack通知）でWebhook URLの取得を教えるが、公開リポジトリにコミットするリスクへの注意がない。L29のconfig.jsonにURLを平文保存する指示も同様
- **修正案**: L28に「Webhook URLは絶対にGitHubの公開リポジトリにコミットしないでください。`.env`ファイルに保存して`.gitignore`に追加しましょう」を追加。L29も同様の注意を追記

## カリキュラム全体の総評（教育設計エージェントC）

### 強み
1. **一貫したデザイン言語** — 全30レッスンでUI/UXパターン（ポイントボックス、ステップ表示、ターミナル風コードブロック、良い例/悪い例の対比）が統一
2. **「Claude Codeへの依頼例」が豊富** — そのまま使えるプロンプト例が実用的
3. **段階的な設計思想** — Ch.1→Ch.6の流れが論理的で初心者が迷わない
4. **Claude Codeの内部動作の可視化** — Read→Edit→Writeのフロー表示が教育的に優秀

### 特に優れているレッスン
- **LessonContent11（プロンプト設計の原則）** — 5原則+テンプレートがコース全体の核心スキル
- **LessonContent19（プロジェクト設計）** — 「一括 vs 段階的」の対比が秀逸
- **LessonContent22（バグ修正）** — エラーメッセージの分解が教育的

### 最も改善が必要なレッスン
